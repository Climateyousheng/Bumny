Windows directory
Application-Specific directory that contains window definition files, the Navigation Tree control file nav.spec and the specification of the navigation buttons nav.buttons.
Application-Specific
Used to distinguish components or files specific to a GHUI-based user interface application from the generic GHUI components.
Window definition file
Held in the windows directory. Each GHUI input panel is defined by up to three types of file. The main file has a .pan extension. Within its header it will define a skeleton file which describes the general format of the panel. The main file may also incorporate one or more window include files. Include files are used when the same or similar sets of questions are used in more than one place.
nav.spec
Control file that describes the Navigation Tree.
nav.buttons
Control file that describes the buttons displayed along the bottom of the Navigation Window.
Navigation Tree
Expanding and collapsing tree widget that represents the hierarchy of categories and subcategories of Input Panels. Any input panels related to the currently selected node are listed in the panel to the right of the tree.
Input panel
A form that displays questions along with input components such as entry boxes, radio buttons and tables. Each component is related to one of the variables in the job. Panels are defined by the windows definition files.
Basis database
A basis database is a text file containing the values of all data items declared in the related variable registers. The format of a basis database matches that of a FORTRAN namelist file. There are three types of database matching to each of the types of variable register: Job basis databases matches to the var.register file. There is one job basis database for each job stored in the application database; ie. each one contains all the user inputs for a particular job. System database matches to the system.register file and is held in a file called system.database. It contains values of constants used in an application. Parameter database matches to the system.register file and is held in a file called parameter.database. It is intended for values of integer constants used to declare sizes of arrays in the system and variable registers.
Variable Registers Variable registers contain declarations for the data objects used in the application. An application requires three registers to be held in the variables application directory: The var.register, system.register, and parameter.register. Each variable register matches to a different type of basis database. Items in the var.register file relate to a particular job setup. The values of these items for a particular job setup are stored in a job basis database in the central database. Usually, items in the var.register file relate the user interface questions. The system.register file contains a mixture of constants, parameters or lists of data, and variables that are hidden from the user such as link variables. The values of items in the system.register are stored in the system.database. The parameter.register file contains constants that can be used in the declaration of variables in the other two registers; usually declarations of array size. The values of items in the parameter.register are stored in the parameter.database. All files but the job basis databases are held in the variables application directory.
